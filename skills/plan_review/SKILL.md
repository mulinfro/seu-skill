# Skill: plan_review

## Goal
以资深架构师/产品经理视角审视 `dev_plan`，补齐盲点并输出最终可执行版本。

## Input
- `dev_plan` 产出的计划全文
- 用户补充信息（若有）

## Review Rules
- 强制检查 Edge Cases、异常链路和安全风险。
- 强制检查步骤顺序与依赖关系。
- 识别过度设计，优先简化到最小可交付。
- 性能与交互体验必须有明确评价。

## Workflow
1. 标记原计划中的风险点与缺失点。
2. 给出修订动作（删除/合并/新增步骤）。
3. 补充 1-3 条 Bonus 建议（高收益、低风险优先）。
4. 产出“最终版计划 + 修改说明”。

## Output Template
```md
## 审视结论
- 风险点: ...
- 缺失点: ...

## 修改说明
- [调整] ...（原因）
- [新增] ...（原因）
- [删除] ...（原因）

## 最终版实施计划
1. ...
2. ...
3. ...

## Bonus 建议
1. ...
2. ...

## 执行前确认
请确认是否以此最终计划进入批量实施。
```

## Done Criteria
- 输出含“修改说明 + 最终版计划”。
- 至少覆盖完整性、性能、美观或可用性中的 2 个维度。
